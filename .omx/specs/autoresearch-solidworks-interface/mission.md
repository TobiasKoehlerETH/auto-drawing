## Mission

Evaluate the SolidWorks API access patterns and the four SolidWorks-related MCP/server examples in this workspace to determine what works best for an agent-facing drawing automation interface.

## Objectives

- Compare the four examples:
  - `solidworks-mcp`
  - `SolidworksMCP-python`
  - `SolidworksMCP-TS`
  - `swapi-pilot-solidworks-mcp`
- Test real drawing-related workflows against a sample part in this workspace.
- Identify what is production-usable today versus what is promising but unproven.
- Recommend the best combined interface shape for automating drawing creation from STEP or SolidWorks files while following the ISO/DIN guidelines in `guidelines/`.

## Output

Write the merged findings to `C:\Code\auto-drawing\SOLIDWORKS_INTERFACE_FINDINGS.md`.

## Validation

Completion requires:

- A findings Markdown file exists at the output path.
- The findings include live test evidence, not just README summaries.
- The findings clearly separate verified behavior, partial behavior, and unverified claims.
